      Update your Uber Ride Count and Distance program from Lab 10 to add the following features (or write a new program that does the following):

      1. Calculate the average velocity for each ride:

        • you can calculate the average velocity using any of the following methods (the results will be different depending on which method you choose):
          • $velocity_x = {end\_position_x - start\_position_x}/{end\_time_x - start\_time_x}$, $velocity_y = {end\_position_y - start\_position_y}/{end\_time_y - start\_time_y}$, $average\_velocity = √(velocity_x^2+velocity_y^2)$
          • $velocity_x = {velocity_{x,1} + velocity_{x,2} + ... + velocity_{x,n}} / n$, $velocity_{x,i} = {Δdistance}_{x,i} / {Δduration}_{x,i}$,
            $velocity_y = {velocity_{y,1} + velocity_{y,2} + ... + velocity_{y,n}} / n$, $velocity_{y,i} = {Δdistance}_{y,i} / {Δduration}_{y,i}$,
            $average\_velocity = √(velocity_x^2+velocity_y^2)$
          • $velocity_i = √{({Δdistance}_{x,i})^2 + (Δdistance_{y,i})^2} / {Δduration_i}$, $average\_velocity = {velocity_1 + velocity_2 + ... + velocity_n} / n$
        • Report velecoty as distance per hour
          • Regardless of the method you choose above to compute velocity, when computing $Δduration = end\_time - start\_time$, do so in hours
          • Do NOT ignore the minutes and seconds data
          • For example, if the time difference is 1 hour, 15 minutes and 12 seconds, then $Δduration = 1.2533$ not 1
        • Feel free to use the sqrt function from the math.h library
          • How to use the sqrt: http://www.cplusplus.com/reference/cmath/sqrt/
          • How to compile with the math.h library: http://stackoverflow.com/questions/4120592/c-programming-sqrt-function
        • Or you can write your own sqrt function
            Use at your own risk:
          • http://en.wikipedia.org/wiki/Methods_of_computing_square_roots
          • http://stackoverflow.com/questions/1623375/writing-your-own-square-root-function

      2. Generate a list of velocities sorted with the fastest ride on top:

        • output to a file a list of ride IDs with their average velocity
        • the ride IDs their average velocity are sorted with the fastest ride on top
